# Contributing

Thank you for your interest in improving the Vuetify ecosystem!

## Submitting Feedback

The easiest way to submit feedback is through [issues.vuetifyjs.com](https://issues.vuetifyjs.com). Select the appropriate project and follow the guided submission process.

Alternatively, you can [create an issue directly](https://github.com/vuetifyjs/ecosystem/issues/new) with the following information:

### Bug Reports

- Project name (Play, Bin, Studio, Snips, Link, Issues)
- Steps to reproduce
- Expected behavior
- Actual behavior
- Browser and OS information
- Screenshots if applicable

### Feature Requests

- Project name
- Problem you're trying to solve
- Proposed solution
- Alternatives considered

## Issue Labels

Issues are labeled by project:

| Label | Project |
|-------|---------|
| `play` | play.vuetifyjs.com |
| `bin` | bin.vuetifyjs.com |
| `studio` | studio.vuetifyjs.com |
| `snips` | snips.vuetifyjs.com |
| `link` | link.vuetifyjs.com |
| `issues` | issues.vuetifyjs.com |

## Code of Conduct

All interactions must follow the [Vuetify Code of Conduct](https://vuetifyjs.com/en/about/code-of-conduct/).
